# Weekly Tasks Orchestrator

Automatically run all three weekly tasks in sequence:

## Week definition

"Current week" means the ISO 8601 week (Monday–Friday) that contains **today's date**, interpreted in the **Europe/Amsterdam** timezone. If the job fires on a Saturday or Sunday (e.g. machine was asleep on Friday), the current week is still the week that just ended — do not skip it.

## Commands to execute (in order)

1. `/hour-registration:weekly-tasks:1. list-weekly-hours-calendar`
   - Fetches Mon-Fri events from Google Calendar for the current week
   - Applies location rules (Eneco, National Holiday, Leave, Vacation, Utrecht)
   - Saves to `hour-registration/data/worked_hours.json`

2. `/hour-registration:weekly-tasks:2. add-weekly-hours-eneco`
   - Reads from `hour-registration/data/worked_hours.json`
   - Submits hours to Eneco Fieldglass

3. `/hour-registration:weekly-tasks:3. add-weekly-hours-sourcelabs`
   - Reads from `hour-registration/data/worked_hours.json`
   - Submits hours to Sourcelabs Administratie

## Schedule

Run **every Friday before end of day**.

All three commands run automatically in sequence without pausing or requiring verification.

## Error handling

If any step fails:

1. **Stop immediately** — do not run subsequent steps.
2. **Report clearly**: which step failed, what the error was, and whether any partial data was written.
3. **How to resume**: fix the reported issue, then re-run only the failed step (or re-run the orchestrator — duplicate-submission guards in steps 2 and 3 prevent double-posting).

Transient failures (network timeout, page load error): the individual step retries up to **2 times** with a 30-second wait before reporting failure.
