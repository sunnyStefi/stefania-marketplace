---
name: 3. add-weekly-hours-sourcelabs
description: "Add the current week's worked hours to the Sourcelabs Administratie tool."
---

Use the Playwright MCP server to navigate to https://app-administration-app.azurewebsites.net/time-tracking.

Read credentials from the macOS Keychain:

- username: `security find-generic-password -s worksheet-sourcelabs -a username -w`
- password: `security find-generic-password -s worksheet-sourcelabs -a password -w`

## 1. Verify the stored week matches the current week

Read `hour-registration/data/worked_hours.json`. Compute the current ISO week (e.g. `2026-W20`) in the **Europe/Amsterdam** timezone and compare it to the `week` field in the file. If they differ, **stop and report the mismatch** — do not submit stale hours.

## 2. Check whether this week's hours were already submitted

Navigate to the current week in the time-tracking view. If any hours are already entered for the week (any project row shows non-zero hours, or the week is marked as submitted/locked), **stop and report that the week was already submitted** — do not add duplicate entries.

## 3. Enter hours

For each day in `worked_hours.json.days`, enter hours according to the **location rules** (Sourcelabs project, hours, and Thuiswerken Vergoeding columns).

When done, click **Indienen** to submit the timesheet.

## Error handling

- **Transient error** (network timeout, page crash): clear any partially entered fields, wait 30 seconds, and retry the full entry from scratch. Retry at most **2 times** before reporting failure.
- **Unknown location value**: stop immediately and report which day has an unrecognized location — do not enter partial hours.
- **Login failure**: stop and report — do not retry credentials automatically.
