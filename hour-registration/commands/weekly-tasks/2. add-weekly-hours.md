---
name: 2. add-weekly-hours
description: "Add the current week's worked hours to both Eneco Fieldglass and the Sourcelabs Administratie tool."
---

Use the Playwright MCP server for both submissions below.

## 0. Shared pre-check (do once, before either submission)

Read `hour-registration/data/worked_hours.json`. Compute the current ISO week (e.g. `2026-W20`) in the **Europe/Amsterdam** timezone and compare it to the `week` field in the file. If they differ, **stop and report the mismatch** — do not submit to either system.

---

## A. Eneco Fieldglass

Navigate to https://www.eu.fieldglass.cloud.sap/time_sheet_list.do?cf=1.

Read credentials from the macOS Keychain:

- username: `security find-generic-password -s worksheet-fieldglass -a username -w`
- password: `security find-generic-password -s worksheet-fieldglass -a password -w`

**Check**: On the timesheet list, locate the row for the current week. If its status is anything other than `Draft` / `Not submitted` (e.g. `Pending Approval`, `Submitted`, `Approved`), **skip this submission and report it** — do not re-open or re-submit.

**Enter hours**: For each day in `worked_hours.json.days`, enter hours according to the **location rules** (Eneco hours column). When done, click **Indienen** to submit.

---

## B. Sourcelabs Administratie

Navigate to https://app-administration-app.azurewebsites.net/time-tracking.

Read credentials from the macOS Keychain:

- username: `security find-generic-password -s worksheet-sourcelabs -a username -w`
- password: `security find-generic-password -s worksheet-sourcelabs -a password -w`

**Check**: Navigate to the current week. If any hours are already entered (any project row shows non-zero hours, or the week is locked/submitted), **skip this submission and report it** — do not add duplicate entries.

**Enter hours**: For each day in `worked_hours.json.days`, enter hours according to the **location rules** (Sourcelabs project, hours, and Thuiswerken Vergoeding columns). When done, click **Indienen** to submit.

---

## Error handling (applies to both)

- **Transient error** (network timeout, page crash): clear any partially entered fields, wait 30 seconds, and retry the full entry from scratch. Retry at most **2 times** before reporting failure.
- **Unknown location value**: stop immediately and report which day has an unrecognized location — do not enter partial hours.
- **Login failure**: stop and report — do not retry credentials automatically.
